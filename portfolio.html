F&B Business Intelligence Platform

End-to-end ELT pipeline on a Modern Data Stack, processing 270K+ sales transactions for a simulated 20-store F&B chain — from raw ingestion to Star Schema warehouse to decision-ready data marts.

  • Built an end-to-end ELT pipeline ingesting data from a REST API (JSON), CSVs, and Google Sheets via n8n/Airflow into PostgreSQL
  • Designed a Star Schema Data Warehouse (1 fact + 5 dimensions) in dbt; converted the fact table from full-refresh to an incremental model (merge strategy + watermark), cutting processing time 3.7x
  • Implemented SCD Type 2 via dbt snapshots for customer history tracking; enforced data contracts and wrote 27 automated tests, including custom reconciliation tests
  • Built 5 analytics-ready data marts powering business dashboards: RFM customer segmentation, inventory forecasting, market basket analysis, and store/employee performance
  • Implemented role-based access control and PII masking for customer data; tuned query performance with indexing, validated via EXPLAIN ANALYZE (~2x improvement)
  • Set up CI/CD (GitHub Actions: lint + Slim CI) and a production-grade Airflow DAG (retries, SLA monitoring, alerting) for orchestration

Modern Retail Data Warehouse (ELT Pipeline)

Automated ELT pipeline unifying 4 heterogeneous retail data sources into a dbt-modeled Star Schema warehouse, powering Metabase dashboards for both executives and operations teams.

success

WWI Retail Data Warehouse — Modern Data Stack

Production-minded ELT pipeline built on Wide World Importers data, unifying 6 source files into a Star Schema warehouse in dbt and shipping 3 Metabase dashboards for Executives, Finance, and Operations.

  • Designed an automated n8n EL workflow ingesting 6 CSV sources (customers, products, salespeople, invoices, order fulfillment, transactions) into PostgreSQL on a 6-hour schedule, with TRUNCATE-based idempotency and primary-key constraints to fail fast on duplicate loads
  • Modeled a Star Schema in dbt (4 dimensions + 5 fact tables) and built 5 downstream data marts powering 3 Metabase dashboards for Executives, Finance, and Operations
  • Implemented SCD Type 2 via dbt snapshots to preserve historical customer attribute changes, and converted 3 detail fact tables from full-refresh to incremental models (delete+insert), cutting no-op rebuild time to under 1 second
  • Added dbt source freshness checks (warn >8h, error >24h) and dbt exposures to map dashboard dependencies before schema changes
  • Built a CI/CD pipeline (GitHub Actions) running dbt build, tests, and freshness checks on every PR against a temporary Postgres instance
